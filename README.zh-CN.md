# daf-wails-template

语言切换： [English](README.md) | [中文](README.zh-CN.md)

> 一个用于生产环境的 Wails 桌面应用模版。\
> 基于 Go 1.23 和 Wails v2.11.0 构建。

------------------------------------------------------------------------

## 🚀 为什么选择 daf-wails-template？

官方的 Wails 模板适合入门，但真实桌面应用经常需要更多基础设施：

- 结构化日志
- 构建时版本注入
- 系统元数据管理
- 用户配置持久化
- 国际化 (i18n)
- 清晰的前后端分离
- 最小可用的生产就绪 UI
- 自定义标题栏支持

该模版将这些能力整理成可复用的架构，让你关注产品而不是重建基础设施。

## 📂 项目结构（一级概览）

仓库顶层目录及用途简要说明：

- `backend/`：Go 后端代码与 Wails 绑定（应用入口、桥接代码、适配器等）。
- `core/`：核心模块，如配置管理、默认值与共享工具。
- `config/`：仓库级配置文件与模板。
- `build/`：构建产物、打包脚本与 CI 辅助。
- `cmd/`：CLI 入口或辅助命令。
- `frontend/`：React + Vite 前端源码、组件、页面与资源。
- `wails.json` 与 `main.go`：Wails 配置与用于嵌入前端资源的 Go 程序入口。
- `wailsjs/`：生成的 Wails JS 运行时绑定，供前端调用。
- `openspec/`：设计提案、规范与变更追踪。
- `docs/`：文档与开发指南。
- `scripts/`：构建或打包的辅助脚本（如 macOS / Windows 构建脚本）。
- `logger/`、`systeminfo/`、`sysconfig/`、`updater/`、`plugin/`、`diagnostics/`：后端的功能模块。
- `package.json`、`pnpm-lock.yaml`：前端包清单与锁文件。

## ⚡ 快速开始

### 1️⃣ 安装 Wails CLI

```bash
go install github.com/wailsapp/wails/v2/cmd/wails@latest
```

### 2️⃣ 安装依赖

```bash
pnpm install
```

### 3️⃣ 开发模式运行

```bash
wails dev
```

### 4️⃣ 构建生产可执行文件

```bash
wails build
```
