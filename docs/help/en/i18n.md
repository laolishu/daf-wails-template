# Internationalization

## Purpose
I18n is driven by app config and the frontend message catalog:
- App config stores the language preference (`i18n.language`).
- Frontend strings live in `frontend/src/i18n.ts`, currently `zh-CN` and `en-US`.
- Backend exposes language read/update APIs for the settings page.

## Usage
- Read or update language via `config.Manager`.
- Expose `App.GetLanguage`/`App.SetLanguage` to the frontend.
- Frontend uses `normalizeLanguage` and `messages` to switch copy.

## Key Fields and APIs
- Config key: `i18n.language` (default `zh-CN`, supports `en-US`)
- Backend APIs: `App.GetLanguage()`, `App.SetLanguage(language)`
- Frontend entry: `frontend/src/i18n.ts`
