## 新增需求
### 需求：开发者帮助文档入口
系统必须在 docs/help/README.md 提供开发者帮助总纲，包含对配置、日志、国际化、远程升级模块的简要说明，并链接到对应的详细文档页。

#### 场景：开发者查找模块入口
- **当** 开发者进入 docs/help 目录
- **那么** 必须能够在 README.md 中看到模块总览与到详细页的链接

### 需求：模块帮助文档结构
系统必须为配置、日志、国际化、远程升级模块分别提供详细文档页，且每个页面至少包含：功能定位、主要使用方式、关键配置或接口说明、相关文档链接。

#### 场景：开发者阅读模块细节
- **当** 开发者打开任一模块帮助页
- **那么** 必须能够快速了解模块作用、如何使用以及可延伸阅读的文档入口

### 需求：面向开发者的表述与语言
系统必须以开发者视角编写帮助文档，语言为中文，避免面向终端用户的操作说明。

#### 场景：开发者阅读帮助文档
- **当** 开发者阅读 docs/help 下的帮助内容
- **那么** 文档必须以开发者为目标读者并使用中文描述
