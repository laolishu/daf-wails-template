# Design Doc：Examples 页面

## 目标
提供一个工具型、简洁的 Examples 页面，用于验证前端与 Wails API 的真实调用链路。

## 设计要点
1. **页面入口**
   - Examples 作为主导航项，名称支持 i18n，默认英文 Examples。

2. **示例卡片结构**
   - 每个示例为独立卡片，包含标题、说明、按钮与结果状态。
   - 结果状态即时展示，避免长文本教学说明。

3. **真实调用约束**
   - Logger：调用 Go API 写入 INFO 日志，固定前缀与时间戳。
   - System Info：读取版本号、构建时间、运行平台（只读）。
   - Config Read：读取当前配置（语言、日志级别）。

4. **操作约束**
   - 不使用 mock，不修改用户数据，不自动修复。
   - 操作结果立即可见。

## 影响范围
- frontend：新增页面与服务调用
- backend：新增日志写入 API（若不存在）
