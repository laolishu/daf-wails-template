## 新增需求
### 需求：升级页检查与下载交互
升级页面必须提供“检查更新”主操作按钮，点击后请求后端升级检查 API，并在弹出页展示升级结果，允许用户选择下载或取消。

#### 场景：检查更新并展示结果
- **当** 用户点击“检查更新”
- **那么** 前端必须调用后端升级检查 API 并展示升级结果弹出页

#### 场景：选择下载更新
- **当** 用户在弹出页选择下载
- **那么** 前端必须调用后端下载/安装 API 并展示执行结果

#### 场景：取消下载
- **当** 用户在弹出页选择取消
- **那么** 前端必须关闭弹出页且不触发下载

### 需求：升级结果展示字段
升级结果弹出页必须展示以下字段（存在时）：当前版本、最新版本、是否强更、发布说明、下载地址。

#### 场景：展示升级字段
- **当** 升级检查返回数据
- **那么** 弹出页必须展示要求字段
