## 新增需求
### 需求：升级关键路径日志
系统必须在 core/updater 关键流程中输出 slog 结构化日志，覆盖升级检查、策略决策、下载、校验与安装步骤。

#### 场景：执行升级流程
- **当** Updater 执行 Run() 流程
- **那么** 必须记录每个步骤的开始、成功与失败日志

### 需求：外部请求日志
系统必须在对外请求交互（升级检查与下载）中记录请求与响应状态，包含至少 endpoint/URL、HTTP 状态码与耗时。

#### 场景：发起升级检查
- **当** HTTPProvider 发起升级检查请求
- **那么** 必须记录请求 endpoint、状态码、耗时与错误原因

#### 场景：下载升级包
- **当** BasicInstaller 下载升级包
- **那么** 必须记录下载 URL、状态码、大小与耗时

### 需求：错误日志与上下文
系统必须在失败路径输出错误日志并附带关键上下文字段（如版本、通道、校验值、文件路径）。

#### 场景：升级失败
- **当** 升级流程中任一步骤失败
- **那么** 必须记录错误原因与可用于排障的上下文字段
