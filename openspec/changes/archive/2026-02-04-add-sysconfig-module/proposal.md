# 变更：添加系统配置模块

## 为什么
系统需要一套标准化的机制来管理编译时确定、运行时只读的全局系统变量（如版本号、构建时间、配置文件路径等）。这些变量应该通过 Go 编译时 ldflags 机制注入，与运行时应用配置（config 模块）相区分。

## 变更内容
- 新增"系统配置模块"（core/sysconfig），管理编译时注入的全局系统变量。
- 明确系统配置与应用配置的边界与使用场景。
- 定义常见系统变量的标准集合（版本、构建时间、配置文件路径、数据目录等）。

## 影响
- 受影响规范：sysconfig
- 受影响代码：core/sysconfig/ 新增、cmd/desktop/main.go 模板更新、构建脚本参考
- 与 core/config 的关系：sysconfig 是运行时只读的系统级信息，config 是用户可修改的应用配置
