## 上下文
DAF-Wails 项目已定义了 sysconfig 模块用于管理系统级配置信息，但当前的构建脚本未充分利用这一机制。开发者需要一套便利的、符合多平台的构建方法，能够自动注入版本、构建时间等信息。

## 目标 / 非目标
- 目标：为所有平台（Linux、Windows、macOS）的构建脚本添加 sysconfig ldflags 注入。
- 目标：保持脚本简洁，支持参数化配置（版本、构建时间、Git 提交、配置目录）。
- 目标：提供一致的用户体验，无论是使用 shell 还是 batch 脚本。
- 非目标：不修改 Wails 框架本身或其配置文件格式；不涉及运行时配置动态注入。

## 决策
- **Wails 集成**：利用 wails build 的 `-ldflags` 参数（如果支持）或通过环境变量传递给 Go 编译器。
- **参数来源**：优先从环境变量或命令行参数获取，其次使用时间戳和 Git 信息的自动化方式。
- **配置目录**：允许通过参数指定，默认值由用户/脚本决定。
- **跨平台**：shell 脚本（Unix/Linux/macOS）与 batch 脚本（Windows）分别实现，逻辑一致。

## 风险 / 权衡
- Wails 可能不支持直接传递 ldflags → 需要调查 Wails build 命令的参数支持，或回退到直接使用 Go 编译。
- 复杂的参数处理会增加脚本维护负担 → 采用默认合理值，支持可选的参数覆盖。

## 迁移计划
- 先更新脚本，无需修改现有项目代码。
- 新项目可从这些脚本开始，开发者无需额外配置即可获得自动注入。
