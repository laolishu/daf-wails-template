# Design Doc：桌面窗口尺寸与布局规范

## 目标
在不改变现有技术栈的前提下，统一桌面应用窗口尺寸与 UI 布局的硬性约束，确保视觉一致性与可维护性。

## 设计要点
1. **窗口尺寸**
   - 初始化默认尺寸与最小尺寸属于应用启动层配置（Wails 窗口参数）。
   - 规范需要明确“不得强制最大化”。

2. **Sidebar 与内容区布局**
   - Sidebar 固定宽度 240px；Logo 区固定高度 64px；菜单项高度与图标尺寸为区间约束。
   - 主内容区宽度为 `窗口宽度 - 240px`；顶部标题栏高度为 56–64px。

3. **缩放与排版约束**
   - 125% 缩放下 UI 不溢出，意味着关键布局不依赖像素级溢出。
   - 中文文本不换行挤压，要求关键标题与菜单文本保持单行展示（必要时通过宽度/字距或缩略策略保障）。

## 影响范围
- **前端**：AppShell 布局参数与样式约束
- **后端/窗口层**：Wails 初始化窗口大小与最小尺寸配置

## 兼容性
- 与现有 AppShell 结构兼容，仅补充硬性尺寸与排版约束。
