## 上下文
Settings 目前为占位页，System Info 为独立页面，且前端缺乏统一的语言切换入口。

## 目标 / 非目标
- 目标：在 Settings 中集中配置项，语言切换即时生效，并持久化到用户配置
- 非目标：引入复杂的 i18n 框架或多语言包体系扩展（仅中/英切换）

## 决策
- 通过后端 API 读取/写入 `i18n.language` 配置项
- 前端在 Settings 中提供语言选项，并在切换后立即刷新当前文案
- System Info 信息作为 Settings 中的只读分组呈现

## 考虑的替代方案
- 保留 System Info 独立页面：不利于设置集中，入口分散
- 切换语言后重启应用：体验较差且不满足“立即生效”要求

## 风险 / 权衡
- 立即生效需要前端具备最小 i18n 切换机制，否则需引入轻量方案

## 迁移计划
- 更新导航与 Settings 页面结构
- 增加后端配置 API 与前端服务封装
- 上线后保持用户现有配置兼容

## 待决问题
- 即时切换是否仅影响当前前端文案，后端日志是否同步切换
