## 需求
### 需求：主页面整体结构
主页面必须采用固定 Sidebar 导航栏 + 右侧主内容区的单页焦点结构，禁止出现顶部复杂工具栏。

#### 场景：用户进入主页面
- **当** 用户打开应用主页面
- **那么** 界面必须显示固定 Sidebar 与右侧主内容区，且不出现顶部工具栏

### 需求：Sidebar 导航配置
Sidebar 必须使用深色蓝灰背景，顶部显示应用 Logo 与名称，中部为纵向导航菜单（图标 + 文本），当前选中项高亮显示；底部必须放置“设置 / 帮助 / 升级”等次级入口。Logo 必须使用 public/logo.png，名称必须来自后端用户配置项 window.title；底部次级入口必须对应 pages 目录下的页面。

#### 场景：导航与次级入口展示
- **当** 用户查看 Sidebar
- **那么** 必须看到品牌区、纵向导航与底部次级入口，且当前选中项高亮

#### 场景：品牌区与次级入口来源
- **当** 应用渲染 Sidebar 品牌区与底部入口
- **那么** Logo 必须来自 public/logo.png，名称来自 window.title，且次级入口指向 pages 目录下的页面

### 需求：主内容区布局
主内容区必须使用接近白色的浅色背景，内容整体居中并保持充足左右留白，页面采用“欢迎 / 空状态”结构。

#### 场景：展示主内容区
- **当** 主页面渲染主内容区
- **那么** 背景为浅色、内容居中且两侧留白明显，并呈现欢迎/空状态布局

### 需求：Health Check 空状态页面
Health Check 页面必须为中心化空状态结构，包含插画或占位图、主标题、简短说明文案与单一主操作按钮（Primary CTA），禁止展示表格或复杂数据。

#### 场景：用户打开 Health Check 页面
- **当** 用户进入 Health Check 页面
- **那么** 页面仅展示插画/占位图、标题、说明与一个主 CTA，且不出现表格或复杂数据

### 需求：视觉风格约束
界面必须面向欧美用户，体现可信、专业、工具型风格；禁止炫酷动画或高饱和色彩，优先可读性与稳定感。

#### 场景：审视整体界面风格
- **当** 用户观察界面整体视觉
- **那么** 风格应克制、可信、专业，且无高饱和色或炫酷动画
